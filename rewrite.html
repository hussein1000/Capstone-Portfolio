Original context

The original README explained how to install and run a small server, with commands, environment variables, and debugging steps. It assumed prior knowledge of the terminal and package managers.

Non-technical rewrite (audience: community stakeholders)

Imagine an app that helps local volunteers organize events. We built a small program that runs on a computer or cloud service and keeps track of events, volunteers, and messages. You don't need to know how it was built to use it — here's what it does and how it helps:

What it does

  • Stores event details (time, place, tasks)
  • Lets volunteers sign up and receive messages
  • Makes it easy for organizers to see who is available

Why it matters

Instead of coordinating by scattered email and spreadsheets, the app centralizes information so volunteers save time and organizers can plan reliably. That means fewer missed tasks and more successful events.

How to get started (simple)

  1. Open the site (or we can host it for you).
  2. Create an organizer account and add your event information.
  3. Invite volunteers by sharing a link — they can sign up with just a few clicks.

Need help?

If you'd like, I can host a demo and walk your team through the app in 15–20 minutes.

Notes on the rewrite process

In the rewrite I removed technical jargon (like "npm install" and "environment variables"), focused on user benefits and outcomes, and used steps with clear, non-technical actions. The goal was to reduce barriers for community members or stakeholders who need to understand the project's value without technical detail.
